# God-Museum 网站设计规范

状态：实现基准。

视觉概念原本应由 Image Gen 先生成；当前会话没有可调用的内置 Image Gen，且未获得 CLI fallback 授权。本规范因此采用纯代码原生、排版主导的博物馆设计，并把这一点作为明确偏差保留。

## 产品角色

- GitHub Markdown 是唯一内容源。
- 网站是同一批内容的公共展厅。
- 公开地址是 https://chendahuang.com/god-museum/。
- Cloudflare Pages 独立部署，Worker 只负责精确路径代理。

## 视觉方向

主题是“夜间开放的无主众神殿”。

- 背景：近黑石材，不使用蓝黑 SaaS 色。
- 正文：旧纸白，而非纯白。
- 主强调：暗金铜色。
- 次强调：低饱和朱砂。
- 标题：宋体/古典衬线，具有碑铭感。
- 界面文字：克制的无衬线，保持现代可读性。
- 结构：开放长轴、细线、编号、展签；避免默认卡片网格和巨大圆角容器。
- 图形：同心圆、门洞、放射线和路径作为空间语言；不使用伪古董花纹。

## 字体

- 展览标题：Iowan Old Style、Songti SC、Noto Serif SC、serif。
- 正文：Inter、PingFang SC、Noto Sans CJK SC、sans-serif。
- 引用与原典：正文衬线字体，增加行距，不使用斜体模拟古意。

## 首页

### 第一视口

允许的可见文案：

- God-Museum
- 众神殿
- 一界，多天，万神，共殿。
- 世界只有一个人间，却有许多套天空。
- 进入众神殿
- 查看 GitHub
- 大殿 / 神座 / 文明 / 来源

构图：

- 安静的顶部导航。
- 左侧为巨大标题和短宣言。
- 右侧为十二门同心结构，使用代码原生线条和编号。
- 底部露出下一段“十二座权柄大殿”，形成连续滚动。

### 十二座大殿

使用开放式编号长列表，不做十二张卡片。悬停或键盘聚焦时显示权柄摘要，点击进入完整大殿。

### 已登记神座

雷公、宙斯、索尔、因陀罗沿同一条水平轴排列。相似权柄用细线连接，文化差异保留独立说明。

### 三层正典

原典层、整理层、设定层形成三条不同密度的纵向带。用户能够一眼理解“来源”和“原创”不会混写。

### 文明通道

以一句大字结束主体：“神不是因为权柄相似才相遇，而是因为记住他们的人相遇。”

## 大殿页面

- 顶部显示大殿编号、名称和权柄。
- 已登记神座按同一轴线排列，不做战力排名。
- 下方渲染 Markdown 全文。
- 雷霆殿提供四神快速对照。

## 神明页面

- 左侧：神名、原文名、传统、进入的大殿。
- 中央：Markdown 正文。
- 右侧：原典层 / 整理层 / 设定层目录与 GitHub 原文链接。
- 不把三层做成互相隐藏的标签页；阅读者始终能看见来源边界。

## 搜索

- 搜索索引在构建时从 Markdown 自动生成。
- 支持标题、原文名、别名、传统、权柄和正文关键词。
- 搜索结果显示条目类型、标题、摘要和路径。
- 无远程搜索服务，无数据库，无用户追踪。

## 动效

- 首屏环形结构缓慢呼吸，幅度极小。
- 列表悬停使用横向线条延展。
- 页面进入使用 160–240ms 淡入与微位移。
- 完整尊重 prefers-reduced-motion。

## 响应式

- 桌面以空间和长轴为主。
- 移动端把同心结构降为背景水印，内容保持单列。
- 移动导航只保留必要入口与搜索。
- 任何正文不得小于 16px；原典引用保持可选择、可复制。
