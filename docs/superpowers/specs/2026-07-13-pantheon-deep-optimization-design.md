# God-Museum 深度优化设计

日期：2026-07-13  
状态：待实施  
方向：来源严谨的档案底座 + 可运行的共同世界 + 可持续阅读的数字展厅

## 1. 目标

本轮不推翻“一界，多天，万神，共殿”和三层正典，而是在现有 191 个稳定馆藏节点之上完成一次结构性升级：

1. 让众神殿从原则清楚的档案馆变成具有状态、程序、代价和后果的共同世界；
2. 把希腊、北欧、吠陀从“两位代表神的样板”补成能够独立阅读的文明单元；
3. 把首页、目录、分区首页和条目页从文件陈列改成清晰的参观系统；
4. 保持原典层、整理层和设定层隔离，任何新增原创都不冒充历史传统；
5. 完成后直接提交、推送并部署生产环境。

本轮不新增日本、伊朗、玛雅、西非等新文明，不建立战力、等级、统一法术或最高神王体系，不重做现有夜间博物馆视觉。

## 2. 总体架构

继续保留现有三层正典：

- 原典层：材料实际能够确认的文本、器物、仪式、地点和历史关系；
- 整理层：God-Museum 的编目、关系、版本、权柄和跨文明比较；
- 设定层：众神殿自身的空间、规则、关系和原创事件。

在内容组织上增加两个功能层：

- 运行法则：说明众神殿怎样处理时间、显身、通道、争议、人类和遗忘；
- 原创纪事：用有明确来源锚点的事件检验运行法则，并留下永久后果。

原典、整理、设定仍是内容真值边界；运行法则和原创纪事是设定层内部的组织方式，不构成第四种正典。

## 3. 众神殿运行法则

### 3.1 四层空间

| 层次 | 性质 | 权限边界 |
| --- | --- | --- |
| 人间 | 唯一共享的历史世界 | 文明接触、翻译、遗忘、发掘和重新讲述发生于此 |
| 诸神域 | 各文明独立的神话秩序 | 不能由众神殿统一改写 |
| 众神殿 | 会面、见证、关系和协商空间 | 只能约束殿内行为与通道，不拥有宇宙主权 |
| 神话剧场 | 有来源事件的受限重演空间 | 服从该事件自己的文本和版本边界 |

稳定定义：众神殿连接诸神域，但不位于诸神域之上；它拥有通行规则，没有宇宙主权。

### 3.2 四种时间

- 人间时：大体线性，记录文明接触、材料保存与发现；
- 域时：各诸神域自己的循环、末日、劫运、夜航和神代时间；
- 殿时：只记录谁在何种条件下能够显现和会面；
- 事时：神话剧场或原创纪事暂时建立的事件时间。

众神殿可以同时保存多个时代的面相，但不能因此自由返回或改写那些时代。一个诸神域的末日也不能自动成为所有神系的共同末日。

### 3.3 身份模型

- 神座：一位神在众神殿中的持续身份；
- 面相：有时代、地区、文本或仪式依据的历史表现；
- 显身：某次进入众神殿时，由一个或多个面相形成的当下行动者；
- 回响：只有名字、图像、残片、称号或误译形成的不完整存在。

复合神格只有在历史中形成持续名称、祭祀、形象或独立行动能力时，才能建立独立神座；否则保留为有来源的融合关系。

### 3.4 四类门

1. 历史门：由贸易、迁徙、翻译、帝国和宗教传播形成；
2. 礼请门：神座之间因邀请、盟约、求助或共同问题临时开启；
3. 朱砂门：纯 setting-original 通道，必须登记目的、期限、代价和关闭条件；
4. 裂隙：由误译、伪证、遗忘、身份冲突或受损圣物形成的非许可通道。

历史通道继续由 `03_civilizations/corridors.md` 管理；原创门和裂隙由新的运行法则与纪事管理，禁止把原创会面伪装成历史传播。

### 3.5 权力与争议

跨域行动遵循四原则：

- 属地：本土权威不自动扩展到共殿或他者诸神域；
- 条件：名字、仪式、地点、圣物、邀请和誓言等条件不能省略；
- 代价：跨域行动必须关闭、放弃、承认、损耗或留下可追踪印记；
- 非先例：一次事件中的胜负不形成跨文明永久排名。

共殿可以裁定门是否合规、身份如何登记、原创是否越界、关系类型是否诚实、通道如何封存；不能裁定哪套神话最终真实、哪位神最高或哪种来世适用于所有死者。

争议采用有限六步程序：封门止损、各方陈述、展示来源与条件、受影响者异议、有限处置、异议永久留档。没有常设最高法官，结论只对该门、关系和事件生效。

### 3.6 人类、遗忘与复苏

人类不是神力电池，而是命名者、译者、见证者、发掘者、请求者和继承／拒绝者。人类社群对自己的神名、遗物和亡者拥有陈述权。

记忆状态分为：

1. 明座：材料、关系和通道稳定；
2. 隐座：仍有材料，但实践或语言联系已中断；
3. 眠座：锚点过少，不能稳定显身；
4. 残响：身份尚未确认；
5. 误醒：由伪证、误译或现代拼接形成的错误显身。

复苏至少需要名字或称号、可定位材料或器物、地点或历史语境、当代见证或阅读行为组成的锚链。复苏不能补回材料从未保存的完整人格。

## 4. 原创纪事

新增独立目录：

- `10_chronicles/README.md`
- `10_chronicles/_template.md`

`08_myths` 继续保存有历史来源的传统事件；`10_chronicles` 只保存 God-Museum 原创正典。

每篇纪事必须包含：触发事件、来源锚点、参与者、两条以上冲突规则、各方真正要保护的东西、解决行为、代价、永久变化和明确的 setting-original 声明。

首批三个事件：

1. 《无主之雷》：一幅无年代雷神图像同时唤起雷公、宙斯、索尔和因陀罗；目标是判断面相、传播证据或残响，而不是选出真雷神。
2. 《死者走错的门》：现代错误译注把两套来世程序拼接，导致亡者通道错接；修复必须由死者所属共同体和译者参与。
3. 《重新被念出的名字》：新残片唤醒静默名字，但无法确认其为独立神、称号、抄写错误或旧神面相；结局保留有限显身与争议身份。

事件的最终产物必须是门、面相、关系、盟约或记忆状态的永久变化，而不是战斗胜负。

## 5. 三个文明完整单元

每个文明本轮达到：一篇深度总览、6–8 位核心神、必要的人类行动者、1–2 件神圣对象、1–2 个异类存在、2–3 个结构地点、3–5 个关键事件、至少一条有证据的文明通道，以及明确的不可错误合并问题。

每个新增条目至少包含一个精确原典锚点和一项可靠译本或现代研究；只有晚期资料时标为 `partial`，争议进入明确章节，不用流行文化图像补完。

## 6. 站点与阅读体验

### 6.1 首页

- 保留首屏和十二大殿；
- 增加“选择参观路径”：初次来访、按权柄、按文明、按相遇；
- “已登记神座”改为“馆藏选录”，只展示 8 个跨文明代表条目；
- 动态显示正式神座总数并链接全部神座；
- 保留三层正典和文明通道收尾。

### 6.2 完整目录

正式馆藏与导览规范分开：存在 `entry.data.id` 的内容计入馆藏；无稳定 ID 的 README、规范和支持页单独进入“导览与规范”。

目录增加类型、文明、来源状态三组过滤，与全文搜索同时生效。过滤按钮使用 `aria-pressed`，提供清除筛选、无结果状态，并分别报告馆藏与导览数量。

### 6.3 分区首页

识别 `README` 分区页：面包屑只显示“众神殿 / 分区名”，条目类型显示“馆藏分区”，元信息展示该分区正式馆藏数量，不再把分区首页伪装为普通馆藏。

### 6.4 条目页

- 元信息增加来源节点数量并链接到来源；
- 右侧“文本边界”改为“阅读边界”；
- 对已有的原典层、整理层、设定层、来源标题增加克制的英文辅助标记；
- 设定层使用低饱和朱砂边线，来源列表使用紧凑无衬线排版；
- 不对未明确分层的普通章节强行归类。

### 6.5 移动端

- 正文必须先于整条辅助侧栏；
- 正文前只保留可折叠本页目录，相关大殿、GitHub 和辅助链接放到正文之后；

- 保持表格横向滚动、正文链接下划线和统一 `:focus-visible` 轮廓。

## 7. README 与支持文档

README 从当前大而全的说明压缩为入口文档，保留：核心宣言、30 秒世界模型、三层正典、四条阅读路线、完整设定／馆藏／纪事入口、网站与授权。

馆藏状态、文明完成度和技术部署分别进入：

- `docs/collection-status.md`
- `docs/deployment.md`
- `CATALOG.md`

README 负责让人进入；基石负责让世界成立；目录负责展示规模；纪事负责证明世界能够运行。

## 8. 并行实施边界

实施拆成三个互不覆盖的工作单元：

1. 世界观单元：`00_foundation`、`07_places/pantheon.md`、`03_civilizations/corridors.md`、`10_chronicles` 和旗舰条目设定层；
2. 文明单元：希腊、北欧、吠陀条目、来源库、文明总览和旧文件迁移；
3. 站点单元：`src`、README、目录和支持文档。

主单元负责处理跨单元冲突、全局链接、内容配置、馆藏计数、首页代表项、提交、推送与部署。

## 9. 验收与上线

按用户要求不进行额外浏览器回归、逐条人工测试或专项性能测试。只执行上线所必需的流程：

1. 检查工作区差异和冲突；
2. 执行生产构建，因为静态产物是部署前置条件；
3. 提交全部改动并推送 `main`；
4. 部署 Cloudflare Pages；
5. 如公开路径代理配置未变化，不重复部署 Router Worker。

构建若因稳定 ID、内部链接、类型或静态生成错误失败，必须修复后再部署；不通过跳过构建直接发布旧产物。

## 10. 完成标准

完成后，读者应能回答：

- 神以神座、面相还是显身进入共殿；
- 不同时间体系如何共存；
- 没有历史接触的神为什么能会面；
- 一位神能带多少本土权力进入共殿；
- 错误通道由谁关闭、决定对谁有效；
- 人类除了信仰之外能改变什么；
- 被遗忘的名字如何复苏并可能误醒；
- 一个原创事件结束后，世界永久改变了什么；
- 希腊、北欧、吠陀各自如何形成独立的文明馆藏网络；
- 新读者从首页、目录和条目页如何连续进入下一篇内容。

达到这些条件后，God-Museum 将从严谨的神话档案系统升级为能够持续产生关系、危机、异议和历史的跨文明共同世界。
