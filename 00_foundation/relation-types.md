# 关系类型

跨神系项目最容易出错的地方，是把“看起来像”写成“本来就是”。God-Museum 使用显式关系类型阻止这种滑移。

## 身份关系

### same-identity · 同神异名

同一传统或明确历史语境中，同一神座使用不同名称。需要文本、祭祀或图像证据。

### aspect-of · 同神异相

同一神座在不同时代、地区或文本中的不同表现。面相之间可以互相矛盾。

## 历史关系

### transmission · 传播变体

一个神随翻译、宗教传播或移民进入新文化后发生转化。必须记录传播方向、时期与中间环节。

### historical-identification · 历史认同

历史人物或传统明确用本土神理解外来神。认同不自动等于身份彻底合并。

### syncretism · 神格融合

两个或多个传统在实际历史中形成新的复合神格、祭祀或共同形象。

## 比较关系

### domain-analogue · 权柄类比

两位独立神座共享部分权柄或叙事功能。它只允许比较，不主张共同来源。

### motif-analogue · 母题类比

不同神话包含相似情节、象征或叙事结构。它不主张同一历史事件。

## 神话内部关系

### co-occurrence · 同文并列

两个存在出现在同一段原典中。它只证明文本并置，不自动建立亲缘、主从或身份关系。

### kinship · 亲缘

父母、子女、手足、婚姻或其他谱系关系。必须标明具体面相或文本版本。

### alliance · 盟约

合作、庇护、主从或共同阵营。

### enmity · 敌对

战争、竞争、复仇、封印或持续冲突。

### succession · 继承

王权、神职、圣物或宇宙秩序的继承与取代。

### bearer-of · 持有

神、英雄或其他存在持有、使用或保管某件圣物。需要同时在圣物档案中建立反向关系。

## 原创关系

### setting-original · 设定原创

God-Museum 为创作建立、原典中不存在的关系。必须同时记录：

- 创建它的设定理由；
- 连接了哪些已知材料；
- 它不得被读作历史主张的明确声明。

### invitation-gate · 礼请门

已建档参与者因邀请、求助、见证或谈判临时建立通行。必须记录同意者、目的、可通过对象、期限和关闭条件。

### vermilion-gate · 朱砂门

God-Museum 为历史上未相遇者建立的显式原创通道。必须同时登记来源锚点、必要性、边界、代价和永久变化。

### temporary-covenant · 临时盟约

只对某一事件、对象或期限有效的原创义务。盟约必须说明双方义务、违反后果、退出方式与是否允许继承。

### memory-anchor · 记忆锚点

名字、材料、地点、仪式或当代见证者对某个神座、面相或回响的保存关系。它只支持该锚点能够证明的部分。

### contested-identity · 争议身份

一个名字、称号、图像或复合神格有多个合理归属，当前证据不足以作唯一裁定。该关系要求并列方案和各自证据，不许为了叙事流畅而擅自合并。

### setting-breach · 设定越界

某一显身、圣物、通道或原创行动超出已登记属地、条件或同意范围。必须记录受影响者、止损方式、处置范围和留存异议。

## 最低证据要求

| 关系 | 最低要求 |
| --- | --- |
| same-identity | 直接文本、祭祀或图像证据 |
| aspect-of | 可说明的同一身份延续 |
| transmission | 时间、地点与传播路径 |
| historical-identification | 历史文献中的明确认同 |
| syncretism | 复合名称、祭祀、形象或文本 |
| domain-analogue | 可分别证明双方权柄 |
| motif-analogue | 可分别证明双方叙事 |
| setting-original | 明确的设定层标记 |
| invitation-gate | 同意、目的、期限与关闭条件 |
| vermilion-gate | 来源锚点、原创声明、代价与永久变化 |
| temporary-covenant | 各方义务、违反后果和退出方式 |
| memory-anchor | 可定位的名字、材料、语境或当代见证 |
| contested-identity | 两个以上可说明的身份方案与未决证据 |
| setting-breach | 已登记边界、越界行为、受影响者与止损记录 |
