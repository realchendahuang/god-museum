# 门与文明通道

众神殿中的门不是为了让任何神在任何地点自由传送。门是一段关系获得通行能力后的空间表现，因此每道门都必须能够说明自己的来源、范围和关闭条件。

## 门的最小登记

一道正式门至少记录：

- 门的类型；
- 起点、终点与可抵达的具体边界；
- 开启者、邀请者、承载者或误开者；
- 开启所需的名字、文本、仪式、地点、器物或盟约；
- 允许通过的显身、人类、信使、声音或物品；
- 有效期限和关闭条件；
- 通行代价与可能受影响的人；
- 门关闭后应保留的记录。

## 一、历史门

历史门由人间真实发生的贸易、迁徙、征服、通婚、翻译、宗教传播、地方化或神格融合形成。

它的核心证据不是神话中两个角色看起来相似，而是：

- 可定位的时间与地点；
- 实际承载交流的人群、机构或媒介；
- 名称、形象、仪式或关系中可见的变化；
- 交流中的权力不对称、抵抗、改名和遗忘。

历史门由 [文明通道](../03_civilizations/corridors.md) 管理。它可以证明相遇或理解曾经发生，不自动证明两位神属于同一神座。

## 二、礼请门

礼请门是已建立神座或有资格的人类共同体，因邀请、求助、见证、谈判或履行盟约而临时开启的门。

礼请门必须满足：

1. 发出邀请的一方拥有自己这一端的通行资格；
2. 另一方的显身或见证者明确接受；
3. 邀请的目的与客人能做的事被预先限定；
4. 任何一方都不能借该门取得对方诸神域的进一步通行权；
5. 邀请结束、条件违反或一方撤回同意时，门必须关闭。

礼请门不建立历史传播关系。一次成功会面可以留下临时盟约或互相见证，不能被后来写为古代两个文明“其实早已相通”。

## 三、朱砂门

朱砂门是 God-Museum 纯原创交汇的正式入口。它的朱砂标记表示：这次相遇建立在已知来源边界之上，却没有被宣称为历史事实。

每道朱砂门必须额外记录：

- 为何必须让历史上未相遇的参与者见面；
- 这次会面分别依托哪些已建档权柄、经验或缺口；
- 哪些行动仍然不得发生；
- 门开启后由谁承担代价；
- 什么永久变化允许留下。

朱砂门不能用来回避来源工作。一位尚未建立基本原典档案的神，不会因为需要参与剧情而突然获得完整性格和权力。

## 四、裂隙

裂隙是没有完成合法登记的连接。它可能来自：

- 错误翻译或现代标签把不同概念当成同一个词；
- 伪证、伪造器物或移除语境的图像；
- 同一名字被多个神座或回响同时响应；
- 一件圣物受损、被强行赋予陌生用途或失去持有关系；
- 某个神座的记忆锚点消失；
- 未经同意的跨域行动。

裂隙不默认是恶意攻击。它首先是一项需要止损、见证和查明的失配。关闭裂隙也不得粗暴删除它暴露出的真实历史联系。

## 门的状态

| 状态 | 含义 |
| --- | --- |
| trace（痕迹） | 已有可见关系，尚不足以通行 |
| open（开启） | 条件满足，可在登记范围内通过 |
| contested（争议） | 身份、同意、范围或代价存在未决异议 |
| sealed（封存） | 暂停通行，但保留门的历史与异议 |
| dormant（沉睡） | 锚点仍在，但当前无法稳定开启 |
| closed（结束） | 按登记条件正常完成，只保留纪录 |

## 边界规则

- 门只给予通行资格，不给予统治权。
- 被邀请的显身不得悄然扩大可通过的面相、从属者或武器。
- 一件物品能够过门，不等于它的全部本土效力能在另一端生效。
- 人类翻译者、持有者或社群见证者是开门条件时，不得在事件中被诸神取代。
- 被关闭的门仍保留开启原因、受影响者和关闭代价，不把冲突修饰成从未发生。
